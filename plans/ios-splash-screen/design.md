# iOS Splash Screen — Design

Date: 2026-06-29

## Goal

Replace the iPhone app's default Expo placeholder launch screen with the
CycleWays brand badge, and match the iPhone home-screen app icon to the same
badge — done the proper Expo-config way so both survive `expo prebuild`.

## Background / current state

The Expo app lives in `apps/mobile`. Both `apps/mobile/ios` and
`apps/mobile/android` are **gitignored** — they are generated by
`expo prebuild` (continuous native generation), so they must not be
hand-edited.

The native splash screen is produced by Expo's `expo-splash-screen` config
plugin. In SDK 56 this plugin must be **installed explicitly** — it is not
currently a dependency (it is absent from `node_modules`, and
`@expo/prebuild-config` 56 no longer generates splashes itself). The existing
native splash is therefore stale, using the **default template placeholder**
`apps/mobile/assets/splash-icon.png` (a gray "target / grid" image). As a
result the generated `ios/CycleWays/SplashScreen.storyboard` references a
`SplashScreen` / `SplashScreenLogo` image that has no matching imageset in
`Images.xcassets`, so launch effectively shows a blank/placeholder screen.

The website's brand logo is the **CYCLEWAYS.APP badge** (a mountain-biker
silhouette over a mountain/river scene, with the wordmark). The master asset is
`public/images/cycleways.logo.png` — a 320×320 RGBA (transparent-background)
sticker. The web loading splash (`index.html`) shows it centered on a gray
gradient (`#ededed → #d2d2d2 → #a8a8a8`).

## Decisions

- **Composition:** the CycleWays brand badge logo (from the website), centered.
- **Background:** solid `#ededed` (iOS launch screens can't use gradients; this
  is the lightest stop of the web gradient and keeps the badge's white sticker
  outline visible).
- **Behavior:** static native launch screen only — Expo auto-hides it on the
  first rendered frame. No JS `preventAutoHide`/fade logic.
- **Appearance:** light only (the app is `userInterfaceStyle: "light"`); no dark
  variant.
- **Home-screen app icon:** also the CycleWays badge, replacing the current blue
  chevron (`apps/mobile/assets/icon.png`). iOS app icons cannot be transparent,
  so the badge is composited onto an **opaque `#ededed`** 1024×1024 square (same
  color as the splash) with a small margin so iOS's rounded-corner mask doesn't
  clip the badge.

## Approach

Config-driven via the `expo-splash-screen` plugin in `app.json`. We do **not**
hand-edit `SplashScreen.storyboard` or the asset catalog — those are regenerated
by prebuild.

### Changes

0. **Install the plugin.** Add `expo-splash-screen` (SDK-matched, via
   `npx expo install`) — required in SDK 56 for prebuild to generate the splash.

1. **Brand splash asset.** Create a high-resolution splash logo by upscaling the
   320×320 `public/images/cycleways.logo.png` to ~1024×1024 (preserving
   transparency) and save it as `apps/mobile/assets/splash-icon.png`, replacing
   the current placeholder.

2. **`apps/mobile/app.json`.** Add the `expo-splash-screen` plugin config:
   - `image`: `./assets/splash-icon.png`
   - `backgroundColor`: `#ededed`
   - `imageWidth`: ~180 (display width in pt; logo centered, `contain` resize)

3. **Brand app icon.** Generate an opaque 1024×1024 icon: composite the badge
   (scaled to ~88% with a margin) onto a solid `#ededed` square, flattened to
   remove alpha, saved as `apps/mobile/assets/icon.png` (the path `app.json`'s
   `icon` key already references — no config change needed).

4. **Regenerate native project.** Run `npx expo prebuild -p ios` so the plugin
   rewrites `ios/CycleWays/SplashScreen.storyboard` and generates the
   `SplashScreen` imageset (@1x/@2x/@3x) in `Images.xcassets` from the new
   splash asset, and regenerates the `AppIcon` asset catalog from the new icon.

## Verification

Build/run on the iOS simulator (`npm run ios` in `apps/mobile`) and confirm:
- the badge appears centered on an `#ededed` background at launch,
- it is reasonably crisp,
- there is no blank flash before the badge appears,
- the home-screen app icon is the badge on `#ededed` (not the blue chevron), with
  no transparent corners.

## Tradeoffs / notes

- The only brand master is 320×320. Upscaling to ~1024 px does not add true
  detail, so at large sizes the badge can look slightly soft. Mitigated by a
  moderate `imageWidth` (~180 pt). If a true high-resolution / vector badge
  becomes available later, drop it in as `splash-icon.png` and re-prebuild.
- Android uses the same placeholder today. This design targets iOS only; Android
  can adopt the same config later by widening the plugin config. (The
  `expo-splash-screen` plugin's root props apply to both platforms, so Android's
  splash is repaired as a side effect; the Android adaptive icon is left as-is.)
- The badge includes the "CYCLEWAYS.APP" wordmark, which is small and may be hard
  to read at home-screen app-icon size. This is accepted to keep the icon and
  splash visually identical; a simplified icon-only mark could replace it later.
